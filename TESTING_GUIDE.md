# Mini NPU Simulator 테스트 가이드

## 1. 목적
이 문서는 프로젝트의 요구사항 충족 여부를 직접 검증하기 위한 상세 테스트 절차를 제공합니다.

검증 범위:
- 모드 1 사용자 입력(3x3)
- 모드 2 data.json 일괄 분석
- 입력 오류 복구
- 동점(UNDECIDED) 정책
- 성능 분석 출력 형식
- 결과 요약(총/통과/실패/실패 케이스)

## 2. 사전 준비
### 2.1 실행 환경
- Python 3.8 이상
- 현재 프로젝트 경로에 다음 파일이 존재해야 함
  - main.py
  - data.json

### 2.2 작업 경로 이동
아래 명령으로 프로젝트 폴더로 이동합니다.

    cd /Users/okbaronok23110/Desktop/E1-3

## 3. 빠른 점검(스모크 테스트)
다음 명령으로 모드 2를 즉시 실행해 전체 흐름이 정상인지 확인합니다.

    printf '2\n' | /usr/bin/python3 main.py

최소 확인 항목:
- 필터 로드 완료 메시지 출력(size_5, size_13, size_25)
- 패턴별 점수/판정/PASS-FAIL 출력
- 성능 분석 표 출력(3x3, 5x5, 13x13, 25x25)
- 결과 요약 출력(총 테스트, 통과, 실패)

## 4. 모드 1 상세 테스트
## 4.1 정상 입력 시나리오
아래 입력은 필터 A=Cross, 필터 B=X, 패턴=X 입니다.

    printf '1\n0 1 0\n1 1 1\n0 1 0\n1 0 1\n0 1 0\n1 0 1\n1 0 1\n0 1 0\n1 0 1\n' | /usr/bin/python3 main.py

기대 결과:
- [1-1] 필터 저장 확인 블록이 출력됨
- A 점수, B 점수 출력
- 판정: B
- [4] 성능 분석(3x3) 블록 출력
- 성능 표에 연산 횟수 9 출력

## 4.2 입력 오류 복구 시나리오
필터 A 첫 줄을 일부러 잘못 입력(숫자 2개)합니다.

    printf '1\n0 1\n0 1 0\n1 1 1\n0 1 0\n1 0 1\n0 1 0\n1 0 1\n1 0 1\n0 1 0\n1 0 1\n' | /usr/bin/python3 main.py

기대 결과:
- 입력 형식 오류 메시지 출력
- 다시 입력해 주세요 메시지 출력
- 프로그램이 종료되지 않고 재입력 후 정상 진행

## 4.3 동점(UNDECIDED) 시나리오
A와 B가 사실상 같은 점수를 갖도록 구성해 epsilon 정책을 확인합니다.

예시 입력:
- 필터 A/B를 동일하게 입력
- 패턴은 어떤 값이든 동일 점수 유도 가능

    printf '1\n1 0 0\n0 1 0\n0 0 1\n1 0 0\n0 1 0\n0 0 1\n1 1 1\n1 1 1\n1 1 1\n' | /usr/bin/python3 main.py

기대 결과:
- 판정: 판정 불가
- 동점 기준 안내 문구(|A-B| < 1e-9) 출력

## 5. 모드 2 상세 테스트
## 5.1 기본 실행

    printf '2\n' | /usr/bin/python3 main.py

기대 결과:
- 필터 로드 단계 출력
- 패턴 분석 단계에서 각 케이스별 결과 출력
  - Cross 점수
  - X 점수
  - 판정
  - expected
  - PASS 또는 FAIL
- 성능 분석 표 출력
  - 크기(NxN)
  - 평균 시간(ms)
  - 연산 횟수(N^2)
- 결과 요약 출력
  - 총 테스트
  - 통과
  - 실패
  - 실패 케이스 목록

## 5.2 케이스 단위 FAIL 확인
data.json 내부에 크기 불일치 또는 스키마 문제가 있는 케이스가 있으면, 해당 케이스만 FAIL 처리되어야 합니다.

확인 포인트:
- 특정 케이스에서 FAIL 메시지를 출력해도 프로그램 전체는 계속 진행
- 마지막 요약에서 실패 케이스가 목록화됨

## 6. 성능 분석 확인 기준
## 6.1 모드 1
- 3x3 기준 성능 분석 블록이 별도로 표시됨
- 평균 시간(ms)과 연산 횟수 9가 출력됨

## 6.2 모드 2
- 3x3, 5x5, 13x13, 25x25가 모두 표시됨
- 연산 횟수가 각각 9, 25, 169, 625로 출력됨

## 7. PASS 기준 요약 체크리스트
아래 항목을 모두 만족하면 테스트 통과로 판단합니다.

- 모드 1 정상 입력이 끝까지 실행되고 판정이 출력됨
- 모드 1에서 필터 저장 확인 단계가 출력됨
- 모드 1 입력 오류 시 재입력 유도 동작
- 모드 1 성능 분석 표 출력(평균 시간, 연산 횟수)
- 모드 2에서 필터 로드와 패턴 분석이 전체 실행됨
- 모드 2에서 케이스 단위 FAIL 처리(프로그램 중단 없음)
- 모드 2 성능 분석 표가 요구 크기 전부 포함
- 결과 요약(총/통과/실패/실패 목록) 출력

## 8. 문제 발생 시 점검 순서
1. Python 실행 경로 확인

    /usr/bin/python3 --version

2. 작업 디렉터리 확인

    pwd

3. 필수 파일 존재 확인

    ls -1 main.py data.json

4. JSON 문법 검증

    /usr/bin/python3 -m json.tool data.json > /dev/null && echo JSON_OK

5. 프로그램 기본 실행

    /usr/bin/python3 main.py

## 9. 선택: 실행 로그 파일로 저장
실행 결과를 파일로 남겨 비교하려면 아래처럼 사용합니다.

    printf '2\n' | /usr/bin/python3 main.py | tee test_mode2.log

다시 확인할 때:

    grep -E 'PASS|FAIL|총 테스트|통과:|실패:' test_mode2.log
